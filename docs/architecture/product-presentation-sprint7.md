# Sprint 7 — Support de présentation produit

Objectif :
- présenter Conforméo simplement dans un contexte beta, pilote ou démo
- garder un discours cohérent avec le produit réellement disponible
- éviter un support marketing lourd ou déconnecté du terrain

## Message court

Conforméo aide une entreprise terrain à garder une lecture simple de son activité :
- ce qu'il faut suivre
- ce qu'il faut faire
- ce qu'il faut produire

Le produit relie trois besoins concrets :
- réglementation
- chantier
- facturation

Le tout reste guidé, lisible et utilisable par des non-experts.

## Problème adressé

Aujourd'hui, beaucoup de petites structures terrain cumulent :
- des informations éclatées entre papier, téléphone, fichiers et messagerie
- une séparation trop forte entre le bureau et le terrain
- des suivis difficiles à relire quand il faut agir vite
- des documents utiles mais difficiles à retrouver ou à finaliser simplement

## Solution proposée

Conforméo propose :
- un desktop simple pour piloter, relire et préparer
- un mobile chantier pensé offline-first pour le terrain
- trois modules activables selon le besoin réel
- une lecture plus claire des priorités, documents et suivis utiles

Lecture simple à rappeler pendant la démo :
- un socle Conforméo
- 3 modules activables : `Réglementation`, `Chantier`, `Facturation`
- pas de grille d'offres complexe au stade beta / pilote

Conforméo n'est pas :
- un ERP complet
- un outil comptable expert
- un CRM avancé
- une GED lourde

## Modules principaux à montrer

### 1. Cockpit desktop

À montrer :
- vue d'ensemble
- KPI utiles
- alertes transverses
- actions à faire
- synthèse par module

Message à faire passer :
- l'utilisateur comprend vite la situation de l'entreprise sans écran surchargé

### 2. Module Chantier

À montrer :
- vue par chantier
- signaux utiles
- documents chantier
- coordination légère

Message à faire passer :
- un chantier peut être préparé, relu et partagé sans outil projet complexe

### 3. Documents chantier

À montrer :
- fiche chantier PDF
- plan de prévention simplifié
- ajustement léger avant export
- rattachement à signature et preuves

Message à faire passer :
- les documents utiles existent vite et restent lisibles

### 4. Module Réglementation

À montrer :
- obligations applicables
- sécurité bâtiment
- DUERP simplifié
- export réglementaire PDF

Message à faire passer :
- la réglementation reste abordable et suivable sans jargon expert

### 5. Module Facturation

À montrer :
- clients
- devis
- factures
- lien chantier ↔ devis / facture

Message à faire passer :
- la gestion commerciale simple reste reliée au terrain

## Déroulé de démo recommandé

Format conseillé :
- 10 à 15 minutes
- un seul cas d'usage concret
- peu de clics, peu de détours

Ordre conseillé :
1. partir du cockpit desktop pour poser le contexte
2. ouvrir un chantier et montrer la lecture synthétique
3. montrer les documents chantier utiles et un export PDF
4. montrer la coordination légère sur chantier ou document
5. montrer le lien vers la facturation si le module est activé
6. terminer par le recueil de feedback simple pour la beta ou le pilote

Si une démo mobile est prévue :
1. montrer un chantier disponible sur l'appareil
2. montrer les états de synchronisation visibles
3. rester sur un parcours très court

## Cas d'usage concrets à raconter

Exemples simples :
- préparer une intervention et sortir rapidement un plan de prévention lisible
- retrouver les documents liés à un chantier sans fouiller plusieurs outils
- suivre un point à traiter sur un chantier ou un document sans task manager
- transformer un devis en facture sans ressaisie complète
- relire rapidement les priorités de l'entreprise depuis le cockpit

## Valeur pour une beta ou un pilote

Ce qu'il faut observer :
- est-ce que l'utilisateur comprend vite où regarder
- est-ce qu'il retrouve les bons documents et les bons statuts
- est-ce que le lien bureau ↔ terrain est jugé utile
- est-ce que les parcours les plus fréquents sont suffisamment rassurants

Ce qu'il ne faut pas promettre :
- une couverture exhaustive de tous les cas réglementaires
- une comptabilité complète
- une signature électronique experte
- une gestion documentaire avancée

## Points d'attention actuels

À expliciter en beta :
- les documents chantier restent légers et volontairement simples
- le recueil de feedback reste manuel
- la coordination reste légère, sans notifications ni workflow complexe
- la synchronisation visible est pensée pour rassurer, pas pour exposer un monitoring technique
