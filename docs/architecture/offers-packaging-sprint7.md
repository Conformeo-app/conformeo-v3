# Sprint 7 — Packaging offres / modules clarifié

Objectif :
- rendre la lecture du produit plus claire avant beta ou pilote
- expliquer simplement ce qui est inclus et ce qui dépend d'un module activé
- éviter une grille commerciale lourde ou prématurée

## Lecture retenue

Pour la beta et le pilote, Conforméo se lit de façon simple :
- un `socle Conforméo`
- des `modules activables`
- quelques `configurations types` faciles à expliquer

Le produit ne se présente pas ici comme :
- une grille tarifaire détaillée
- une suite ERP avec dizaines d'options
- des offres marketing `Starter / Pro / Enterprise`

Ce ticket clarifie le périmètre produit. Il ne définit pas une stratégie pricing.

## 1. Socle Conforméo

Le socle correspond à ce qui structure l'expérience globale :
- connexion et contexte multi-organisation
- rôles et permissions simples
- activation des modules par organisation
- cockpit desktop et lectures synthétiques
- audit minimal des actions sensibles
- amélioration du wording, des erreurs visibles et de la lisibilité générale
- support beta léger :
  - check-list beta fermée
  - recueil de feedback simple
  - support de présentation produit

Ce socle sert de base commune. Il ne remplace pas les modules métier.

## 2. Module Réglementation

Le module `Réglementation` couvre aujourd'hui :
- profil entreprise et questionnaire réglementaire simple
- obligations applicables
- sécurité bâtiment
- DUERP simplifié
- preuves réglementaires simples
- export réglementaire PDF

Ce module convient quand l'enjeu principal est :
- comprendre ce qu'il faut suivre
- garder une lecture simple de la conformité utile

## 3. Module Chantier

Le module `Chantier` couvre aujourd'hui :
- vue chantier sur desktop
- lecture terrain mobile-first
- fonctionnement offline-first sur les flux critiques
- preuves terrain, notes, checklist, signalements, signatures
- documents chantier légers
- coordination légère sur chantier et document chantier

Ce module convient quand l'enjeu principal est :
- préparer, suivre et relire les interventions terrain

## 4. Module Facturation

Le module `Facturation` couvre aujourd'hui :
- clients
- devis
- factures
- statuts simples et suivi léger
- paiement simple
- numérotation simple
- PDF devis / facture
- lien chantier ↔ devis / facture

Ce module convient quand l'enjeu principal est :
- garder une facturation simple reliée à l'activité réelle

## 5. Liens utiles entre modules

Le produit est modulaire, mais certains liens apportent la valeur :
- le cockpit relit les informations utiles des modules activés
- le chantier peut porter des documents utiles, des preuves et une coordination légère
- le chantier peut être relié à un devis ou une facture
- la facturation reste plus lisible quand le lien terrain est conservé

## 6. Configurations simples à expliquer en beta / pilote

Plutôt qu'une grille d'offres complexe, la lecture recommandée est :

### Configuration A — Socle + Réglementation

Pour une structure qui veut surtout :
- clarifier ses obligations
- suivre ses éléments réglementaires utiles

### Configuration B — Socle + Chantier

Pour une structure qui veut surtout :
- mieux préparer et relire ses interventions
- disposer d'un usage terrain mobile-first

### Configuration C — Socle + Chantier + Facturation

Pour une structure qui veut surtout :
- relier le terrain, les documents et la facturation simple

### Configuration D — Socle + Réglementation + Chantier + Facturation

Pour une lecture transverse plus complète, tout en restant sur un produit simple.

## 7. Ce qu'il faut dire clairement en démo ou en pilote

À dire :
- les modules sont activables selon le besoin réel
- le cockpit relit ce qui est utile sans tout montrer d'un coup
- le produit reste volontairement simple

À ne pas promettre :
- une tarification définitive détaillée
- un paramétrage commercial complexe
- un ERP complet
- une GED riche
- un moteur réglementaire expert sur tous les cas

## 8. Résumé en une phrase

Conforméo se présente simplement comme :
- un socle commun de pilotage
- avec 3 modules activables : `Réglementation`, `Chantier`, `Facturation`
- et des combinaisons adaptées au besoin réel de la beta ou du pilote
