# Sprint 7 — Recueil de feedback simple

Objectif :
- permettre un retour beta ou pilote sans outil de support complexe
- garder un format court, compréhensible et facile à transmettre

Choix retenu :
- un panneau desktop `Donner un retour`
- des catégories simples :
  - `bloquant`
  - `incompréhension`
  - `amélioration`
  - `retour positif`
- une zone concernée simple :
  - `cockpit`
  - `chantier`
  - `documents chantier`
  - `facturation`
  - `réglementation`
  - `synchronisation visible`
  - `autre`
- un message libre court
- un texte structuré prêt à copier dans le canal beta ou pilote habituel

Pourquoi ce choix :
- pas de backend supplémentaire
- pas de système de tickets
- pas de workflow de support
- un retour facile à transmettre pendant la beta fermée

Ce que le ticket ne fait pas :
- aucune collecte serveur
- aucune notification
- aucune priorisation ou assignation automatique du feedback
